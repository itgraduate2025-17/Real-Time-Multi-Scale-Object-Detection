User Guide

Understanding The Project

The system provides five detection types for versatile use. Image Detection allows users to upload a single image and detect objects with bounding boxes, labels, and distance estimation. Video Detection processes pre-recorded videos to identify objects frame by frame in real-time. Live Detection uses device’s webcam to perform continuous object detection on live camera feed. Multi-Scale Detection enhances accuracy by detecting objects of various sizes within an image and video, ensuring better recognition in complex scenes. Finally, Helmet Detection is specifically designed to identify individuals wearing or not wearing helmets, making it ideal for safety compliance in environments like construction sites.

Project Workflow

Step 1: Launch the web application in your browser.
Step 2: Select the type of input you want to process – Image, Video, or Live Webcam.
Step 3: Click on the "Choose File" button to begin file selection.
Step 4: A popup appears showing the folder path guidelines:
  • Sample images: Placed in static/uploads/images/
  • Sample videos: Placed in static/uploads/videos/
  • Sample images and videos for Multi-Scale Detection: Placed in static/uploads/multiscale/
  • Sample images and videos for Helmet Detection: Stored in static/uploads/helmet/
Step 5: After selecting the file, click the "Upload" button to start the detection process.
Step 6: The system processes the file using Real-Time Multi-scaling Object detection Model Using Deep Learning.
Step 7: Once processing is complete, the detection results will be displayed on the screen with bounding boxes and labels.

Understanding the Detection Results

  • Label: COCO labels appear in lowercase, while custom dataset labels are Capitalized for clarity.
  • Confidence Score: A numeric value (e.g., 0.91) showing how confident the model is about the detection.
  • Distance: Estimated distance in meters, shown in blue text as Distance: 2.5m.

Limitations

  • Smoke Interference: Heavy smoke can obstruct object visibility, reducing detection accuracy.
  • Poor Lighting Conditions: Low or uneven lighting can negatively affect model predictions.
  • Weather Conditions: Rain, fog, or dust can blur the scene and lead to incorrect detections.
  • Hardware Requirements: High-quality camera sensors are essential for accurate distance estimation and detection.
